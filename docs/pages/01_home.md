## 3. 섹션별 상세 기획

### 3-1. Hero

**목적**: 서비스 가치 전달 + 검색 진입

**현재**
- 슬로건: "Discover Korean Beauty Like Never Before"
- 장식 이모지 4개 (분위기용)
- 검색창 UI (입력은 되나 결과 없음)
- 추천 태그: Hyaluronic Acid, Niacinamide, COSRX, Sunscreen SPF50

**개선 사항**
- [ ] 검색창 → Products 또는 Ingredients 통합 검색 연결
- [ ] 검색 결과 드롭다운 (자동완성): 제품명·브랜드·성분명 실시간 매칭
- [ ] 추천 태그를 동적으로 관리 (트렌딩 키워드 반영)
- [ ] Hero 배경에 실제 제품 이미지 또는 그라디언트 애니메이션 적용

**UX 고려사항**
- 모바일에서 검색창이 키보드에 가려지지 않도록 스크롤 처리
- 태그 클릭 시 해당 검색어가 이미 입력된 상태로 결과 페이지 진입

---

### 3-2. Category 탐색

**목적**: 카테고리별 빠른 제품 탐색 진입

**현재**: 6개 카테고리 버튼 → Products 페이지로 이동 (카테고리 필터 미전달)

**개선 사항**
- [ ] 카테고리 클릭 시 Products 페이지의 해당 카테고리 필터가 자동 활성화
- [ ] Sunscreen, Eye Cream 카테고리 제품 데이터 추가 (현재 없음)
- [ ] 카테고리별 제품 수 뱃지 표시 (e.g. "Serum (4)")

---

### 3-3. Trending Now

**목적**: 인기 제품을 바로 노출해 탐색 의욕 유발

**현재**: 하드코딩 4개 카드, 좋아요 토글 (새로고침 시 초기화)

**개선 사항**
- [ ] 좋아요 상태 LocalStorage 저장
- [ ] "View Details" 버튼 → 제품 상세 모달/페이지 연결
- [ ] 트렌딩 기준 표시 (e.g. "이번 주 조회수 기준")
- [ ] 카드 수 4 → 6개로 확장, 가로 스크롤 또는 그리드
- [ ] Phase 2: 실시간 인기순 API 연동

---

### 3-4. Feature 소개

**목적**: 앱의 핵심 기능 3가지를 직관적으로 설명

**현재**: Scan Ingredients / Find Stores / Skin Quiz 3개 카드 (클릭 불가)

**개선 사항**
- [ ] 각 Feature 카드 클릭 시 해당 페이지로 이동
- [ ] 아이콘을 실제 일러스트 또는 SVG 아이콘으로 교체
- [ ] 애니메이션 효과 추가 (스크롤 진입 시 fade-in)

---

### 3-5. Skin Quiz CTA 배너

**현재**: 배너 버튼 클릭 → Skin Quiz 페이지 이동 (정상 동작)

**개선 사항**
- [ ] 퀴즈 완료한 사용자에게는 "내 결과 보기" 문구로 변경
- [ ] 결과 저장 시 퀴즈 완료 상태 유지 (LocalStorage)

## 4. 데이터 요구사항

| 데이터 | 출처 | 비고 |
|---|---|---|
| 트렌딩 제품 | `data/products.json` | Phase 1: 정적, Phase 2: API |
| 카테고리 목록 | 상수 | Products 페이지와 동기화 필요 |
| 검색 인덱스 | 제품명 + 브랜드 + 성분명 | 통합 검색을 위해 병합 필요 |
| 추천 태그 | `data/trending-tags.json` | Phase 1: 정적 |
