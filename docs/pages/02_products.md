# Products 페이지 기획서

> 최종 수정: 2026-04-15

---

## 1. 목적

K-뷰티 제품을 **탐색·비교·발견**하는 메인 카탈로그 페이지.  
카테고리·피부 타입·정렬 필터로 원하는 제품을 빠르게 찾고,  
상세 정보로 구매 결정을 돕는다.

## 3. 섹션별 상세 기획

### 3-1. 제품 히어로 + 검색창

**현재**: 제목 + 설명 + 검색창 (Products 내부 검색, 정상 동작)

**개선 사항**
- [ ] Home 검색과 쿼리 파라미터로 연동 (Home에서 검색 → Products로 이동 시 검색어 유지)
- [ ] 검색 결과 수 실시간 표시 ("12 products found" → 현재는 하단에 표시)

---

### 3-2. 필터 사이드바

**현재 필터 항목**
- Category: All / Toner / Serum / Moisturizer / Cleanser / Sunscreen / Eye Cream
- Skin Type: All Skin / Dry / Oily / Sensitive / Combination / Mature
- Sort By: Most Popular / Highest Rated

**개선 사항**
- [ ] **가격 범위 슬라이더** 추가 (₩0 ~ ₩100,000+)
- [ ] **브랜드 필터** 추가 (다중 선택 체크박스)
- [ ] **Safety 필터** 추가 (Ingredients 페이지와 연계 — 특정 성분 포함/제외)
- [ ] 활성 필터 태그 표시 + 개별 제거 버튼
- [ ] "필터 초기화" 버튼
- [ ] 모바일: 사이드바 → 하단 슬라이드업 필터 시트
- [ ] 필터 상태를 URL 쿼리 파라미터로 저장 (공유·뒤로가기 지원)

---

### 3-3. 제품 그리드

**현재**: 카드형 그리드, 이모지 + 배경색으로 제품 표현

**카드 구성 요소**
- 이모지 + 배경색 (이미지 대용)
- 브랜드명 / 제품명
- 별점 + 리뷰 수
- 피부 타입 뱃지 / 가격
- 좋아요 버튼 / View Details 버튼

**개선 사항**
- [ ] **제품 상세 진입** — View Details 클릭 시 상세 모달 또는 /products/:id 페이지
- [ ] **실제 제품 이미지** 표시 (Phase 2, CDN 호스팅)
- [ ] **리스트 뷰 전환** 옵션 (그리드 ↔ 리스트 토글)
- [ ] **비교 기능** — 체크박스 선택 후 최대 3개 제품 나란히 비교
- [ ] **무한 스크롤 또는 페이지네이션** (데이터 확장 시 필수)
- [ ] 좋아요 상태 LocalStorage 저장

---

### 3-4. 제품 상세 (신규 기획)

View Details 클릭 시 표시할 상세 콘텐츠 구조.

**모달 방식 (Phase 1 권장)**

```
┌─────────────────────────────────┐
│  [제품 이미지 / 이모지]           │
│  브랜드명                        │
│  제품명                          │
│  ⭐ 4.9  (12,400 reviews)       │
│  ₩18,000                        │
│                                  │
│  피부 타입: All Skin             │
│  카테고리: Serum                 │
│                                  │
│  [주요 성분 태그]                │
│  → 성분 클릭 시 Ingredients 이동 │
│                                  │
│  [구매하기] [위시리스트 추가]     │
└─────────────────────────────────┘
```

**페이지 방식 (Phase 2)**
- URL: `/products/:id`
- 성분 목록 전체 표시 + 각 성분 설명 링크
- 유사 제품 추천 섹션
- 사용자 리뷰 섹션

## 4. 데이터 요구사항

### 현재 제품 데이터 구조
```js
{
  name, brand, tag, price, skin,
  category, color, emoji, rating, reviews
}
```

### 목표 데이터 구조 (Phase 2)
```js
{
  id,
  name,
  brand,
  category,
  price_krw,
  price_usd,          // 환율 변환
  skin_types: [],     // 배열
  rating,
  review_count,
  image_url,
  thumbnail_color,    // 이미지 없을 때 fallback
  tags: [],
  ingredients: [],    // Ingredients 페이지와 연동
  description,
  where_to_buy: [],   // Shopping Map과 연동
  is_featured,
  created_at
}
```

## 7. SEO 고려사항 (Phase 2+)

- 각 제품 페이지 고유 URL (`/products/cosrx-snail-96-mucin`)
- 메타 태그: 제품명, 브랜드, 주요 성분 포함
- 구조화 데이터: `Product` 스키마 (가격, 평점, 재고)
- 이미지 alt 텍스트 최적화
